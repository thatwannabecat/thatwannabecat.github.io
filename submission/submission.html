Due to the fact that this is a github page there is an associated github repository with this project. This will allow users to be able to submit changes on their own terms if they feel they have a game that meets the guidelines set.

Whats Required to Submit a Game?

A github account is all you need. There are two main methods to process a submission.

I don't know how to code!

That's ok! You don't need to know how to code to submit a game. Github has an issue system. All you need to do is use your git hub account to submit a new issue and give it the submission label. Provide as much info about the game and characters who are trans in it as possible. An example submission can be found here to give an idea of what is needed.

I do know how to code html!

Great! You can follow this contribution guideline here to get started. All you need to do is add the entry into the index html code as you would a normal table. When making your pull request its best that you provide further detail about the game, and characters in it. When the pull request is created a contributor (currently just me :/ (that_wannabe_cat)) will review the changes to the code and either mark it for approval and merge it, or request changes. If you have cloned the repository on your computer make sure to make pulls reguraly for updates.

What if I disagree with one of the games?

Contributors are humans too. So mistake will be made, more information about the game may be learned, or something that will push for a removal or rexamining of a character. If you want to get a discussion started yourself you can submit a removal issue following this example here. I don't want people making pull requests for removals as thats fairly darn easy to do and a simple issue and 5 minutes by a contributor can solve it.

Whats a Contributor?

Someone who as the power to approve or reject pull requests, and make direct changes to the code directly. Think of them like moderators on a server.
